## 一句话版本

**我们要造一个"AI程序员专用电脑"，让AI能自动写代码、跑回测、出报告——全程无人值守。**

---

## 类比理解

想象你雇了一个**超级实习生**：

- 你给他一张纸条："试试在比特币相关市场上，价格低于0.3时买入，高于0.5时卖出"
- 他走进一间**已经配好所有工具的办公室**（电脑、数据、代码模板）
- 2小时后，他交出一份**完整报告**：代码写得怎么样、赚没赚钱、能不能用

**我们要造的就是这个"办公室 + 实习生管理系统"**

## 具体要造什么

### 1. 工人的工作室（Devbox Agent环境）

| 里面有什么       | 干嘛用的                            |
| ---------------- | ----------------------------------- |
| **历史交易数据** | 工人需要"过去的市场记录"来测试策略  |
| **代码模板**     | 不用从零写，填空就行                |
| **Rust工具链**   | 编译、运行、出结果                  |
| **AGENTS.md**    | 说明书："第一步干嘛、第二步干嘛..." |

### 2. 工头的对讲机系统（Control Plane / devbox CLI）

```bash
# 你（或AI工头）发的指令
devbox start --task "低价买入策略.md"   # 开工
devbox status 001                         # 查进度
devbox snapshot 001                       # 偷看工人在干嘛
devbox steer 001 "这里改一下"              # 中途指导
devbox handoff 001                        # 收成品
```

### 3. 工人的交付物（Artifacts）

工人干完活，桌上必须摆这几样东西：

- `strategy.rs` —— 写的策略代码
- `metrics.json` —— 赚了/亏了多少钱
- `report.md` —— 人话总结："这个策略靠谱/不靠谱，因为..."
- `backtest_events.jsonl` —— 每一笔交易的详细记录

## 为什么不用现成方案？

| 现成方案             | 问题                         | 我们的解法                          |
| -------------------- | ---------------------------- | ----------------------------------- |
| 让AI直接写Python脚本 | 慢、不稳定、和生产环境不一致 | **统一Rust栈**，模板化开发          |
| 人工配置环境         | 每次都要装依赖、调路径       | **预配置Devbox**，开箱即用          |
| SSH进去看AI在干嘛    | 麻烦、不能自动化             | **文件协议 + 状态机**，程序自动监控 |
| 跑完策略没记录       | 不知道AI干了什么             | **强制交付物 + 事件日志**           |

## 当前阶段（PoC）要证明什么

> **"给一个策略描述，2小时后能不能拿到完整报告？"**

具体测试：

1. 启动一个干净的虚拟机（Incus容器）
2. 扔进去一个简单任务："BTC市场，价格<0.3买入，>0.5卖出"
3. AI工人自动：
   - 读说明书（AGENTS.md）
   - 下载数据
   - 改代码
   - 编译（失败了就重试）
   - 跑回测
   - 写报告
4. **人全程不干预，看能不能跑通**

## 技术选型一句话解释

- **Rust**：快、可靠、和生产环境一样（以后能直接上线）
- **tmux**：让AI在后台跑，断了SSH也能继续
- **文件协议**：不用搞复杂消息队列，读写文件最简单可靠
- **Nix/Ansible**：环境配置像公式，100台机器都一样
